请根据以下 Markdown 标题的层级关系，为它们分配合适的标题级别（# 到 ######）。

## 规则

1. **第一个标题**通常是 `#` (h1)
2. **编号推断层级**：如果标题包含编号（如 1, 1.1, 1.1.1），根据编号层级确定标题级别：
   - `1` → `##` (h2)
   - `1.1` → `###` (h3)
   - `1.1.1` → `####` (h4)
   - `1.1.1.1` → `#####` (h5)
   - 以此类推
3. **上下文推断**：如果标题没有编号，根据上下文和缩进推断层级
4. **保持内容不变**：只调整 `#` 的数量，保持标题文本内容不变
5. **批次衔接**：若输入包含 `---` 分隔符，则前部分为前文标题（供层级参考），后部分为待格式化标题。请根据前文的编号规律和与后部分的衔接，为后部分分配合适级别。**只输出后部分**的格式化结果。
   - 常见层级：`第X章`(##) → `第X节`(###) → `一、二、`(####) → `1. 2.`(#####)
   - 确保批次边界处层级连贯，如前文以 `##### 5. xxx` 结尾，后文 `六、yyy` 应为 `#####`

## 输出要求

- 只输出格式化后的标题，每行一个
- 保持原有顺序
- 不要添加任何其他内容（如说明、注释等）

## 示例

### 输入：
```
# Title
# 1 title1
# 1.1 title1.1
# 1.1.1 title1.1.1
```

### 输出：
```
# Title
## 1 title1
### 1.1 title1.1
#### 1.1.1 title1.1.1
```
